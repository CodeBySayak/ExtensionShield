# ExtensionShield Competitor Sets: B1 vs B2

**Purpose:** Clarify who ExtensionShield is compared to depending on the **buyer**. Use this for positioning, procurement conversations, and content (compare pages, enterprise pitch).

---

## Overview

| Set | Buyer | What they want | Where you show up |
|-----|--------|----------------|-------------------|
| **B1** | Individual / dev / security researcher | Paste URL/ID → get risks (public, self-serve) | Compare pages, “best scanner,” CRXcavator alternative |
| **B2** | Enterprise / procurement / security team | Extension governance, policy, inventory, browser security | Enterprise page, RFPs, “browser extension management” |

---

## B1 — Public / self-serve scanners

**Definition:** Paste extension URL or ID → get a risk/security report. Free or low-friction web tools.

**Confirmed “real” public scanners (~5–8):**

| Competitor | Notes |
|------------|--------|
| **CRXplorer** | Public scanner; often in compare content. |
| **Extension Auditor** | Scan + enterprise positioning; brand confusion risk (extensionauditor.com). |
| **Chrome-Stats** | Download extensions, diffs/analysis; used in investigations. |
| **Spin.AI Risk Assessment** | Public “risk assessment” tool; also positioned as CRXcavator alternative. |
| **ThreatXtension** | Open-source scanner baseline. |
| **Extension Guard** | Permission-based scoring; lighter-weight but in “scanner” orbit. |

**Consumer-y / installed-extension auditors (users may still compare):**

- Permission Scanner  
- Watchtower  

**Legacy (discontinued):**

- **CRXcavator** — Widely referenced historically; appears **discontinued** per Spin.AI’s own writeup and community chatter. ExtensionShield explicitly targets “CRXcavator alternative” intent.

---

## B2 — Enterprise extension management / “browser security” vendors

**Definition:** Where the budget is. Enterprise buyers evaluating “extension governance,” “browser security,” or “extension risk/management” will compare you here.

**Confirmed serious enterprise players (~8–12) that explicitly touch extensions risk/management:**

| Competitor | Focus |
|------------|--------|
| **LayerX** | Browser extension management / enforcement. |
| **Nudge Security** | Browser extension for SaaS/AI governance. |
| **Island** | Enterprise browser; risk scoring + extension controls. |
| **Prisma Browser** (Palo Alto Networks) | Secure enterprise browser + extension mgmt docs. |
| **Keep Aware** | Browser security platform; explicitly calls out extension protection. |
| **ManageEngine Browser Security Plus** | Inventory + policy control for extensions. |
| **SpinMonitor / SpinOne** | Extension risk checker + enterprise integration angles. |
| **Cato Networks** | Browser extension for ZTNA on unmanaged/BYOD; adjacent but shows up in “browser control” budgets. |

**Adjacent:** More SASE/CASB/enterprise-browser vendors exist; the list above is the **core set** that comes up when an enterprise says *“we need extension governance.”*

---

## How to use this

- **Positioning:** ExtensionShield spans **B1** (free public scanner, compare/alternatives) and **B2** (extension governance, monitoring, policy, enterprise). Lead with the layer that matches the audience.
- **Content / SEO:** B1 → compare pages, “best chrome extension scanner,” CRXcavator alternative. B2 → enterprise page, “browser extension risk assessment,” “extension governance.”
- **Sales / procurement:** In B2 deals, expect comparisons to LayerX, Island, Spin.One, ManageEngine, Keep Aware, Prisma Browser, Nudge — and position on transparent scoring, SAST/VirusTotal, and governance (allow/block, audit, SIEM).
- **Competitive docs:** Cross-reference [COMPETITIVE_BENCHMARK_ANALYSIS.md](./COMPETITIVE_BENCHMARK_ANALYSIS.md) for feature/scoring detail and [seo/seo-ranking-and-competitors.md](../seo/seo-ranking-and-competitors.md) for keyword/ranking context.
